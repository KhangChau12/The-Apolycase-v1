📋 Plan: Zombie Graphics System Overhaul — Biomechanical Architecture
TL;DR
Thiết kế hệ thống đồ họa toàn diện cho 5 loại zombie với 3 tier, từ prototype polygon đơn giản lên system animation, combo attack, tier morphology đầy đủ. Zombie sẽ là hybrid máy móc + sinh vật với walk cycles, attack choreography, combo tells, và particle auras dựa theo tier.

Recommended approach:

Animation Framework: State machine (Idle, Walk, Attack, Hit, Stun) + keyframe data per archetype/tier
Tier Evolution: Tier 1-3 thay đổi hình dạng (limbs/panels), màu sắc, auras, proportions
Attack System: Wind-up tells + combo chains + per-archetype choreography
Rendering: Refactor từ static polygon sang dynamic compositable limb system
Performance: Per-frame ≤ 2-3 ops/zombie, target 60fps
🎬 Phase 1: Animation State Machine Infrastructure
Goal: Framework animation linh hoạt hỗ trợ walk cycles, attack patterns, idle, transitions.

Create src/data/zombieAnimationData.ts:

AnimationFrame: { duration, bodyScale, limbOffsets, weaponRotation, particleEmit? }
AnimationClip: { name, frames[], loopable, priority }
Per-archetype per-tier clip definitions (Regular: 4-frame walk, Fast: 6-frame walk, etc.)
Extend Zombie.ts animation state:

animationState: 'idle' | 'walk' | 'attack' | 'hit' | 'stun'
currentClip: AnimationClip | null, frameIndex, frameTimer
comboCounter (0-3), windupTimer (attack tell, 0-0.4s)
Implement updateAnimation(dt) → frame progression + auto-transitions + combo logic

🦴 Phase 2 & 4: Tier-Based Morphology + Limb Rendering System
Goal: Tier 1-3 zombies look visually distinct — changed body structure, added tools, unique colors, proportional shifts.

Create src/data/tierMorphology.ts:

Per-tier per-archetype morphology deltas (limb count, gears, panels, weapons)
Tier 1 Regular: +extra limb pair, greenish glow, bristles
Tier 3 Regular: Skeletal limbs, mechanical spine, neon glow, 8 articulated segments
Similar progression for Fast/Tank/Armored/Boss
Create NEW: src/rendering/ZombieLimbs.ts:

LimbSegment interface: { type, relativePos, rotation, scale, color, glow?, children? }
Recursive limb tree (arm → hand → claws)
Types: 'circle', 'rectangle', 'triangle', 'gear', 'plate', 'blade', 'spike'
Build limb skeletons in zombieAnimationData.ts:

LIMB_SKELETONS[archetype][tier] — static hierarchy per variant
Regular tier 0: body + head + 2 arms + 2 legs
Regular tier 3: articulated spine + 8 limbs + mechanical joints
Tank tier 3: heavy body + pillar legs + weapon arm + armor plating
Extend animation frame data:

AnimationFrame.limbDeltas: { limbId: { rotationDelta, scaleDelta, colorShift } }
Example walk cycle: frame 0 = right arm +30°, left leg -30°, etc.
Refactor rendering pipeline:

Replace individual _drawZombieRegular/Fast/... with unified _drawZombieComposite()
Fetch limb skeleton + animation deltas
Recursively draw each limb (shape + children)
Remove old draw methods
⚔️ Phase 3: Attack Animation & Combo System
Goal: Mỗi attack có wind-up tell, combo chain, archetype-specific choreography.

Extend attack logic in Zombie.ts:

Before fire: windupTimer = ARCHETYPE_WINDUP[archetype] (0.2-0.4s)
Render wind-up as scale/pose change (visual cue for player)
After wind-up: fire attack, increment comboCounter (damage multiplier: 1.0x → 1.35x)
Per-archetype combo patterns:

Regular: Claw 1 → Claw 2 → Double bite → Reset
Fast: Dash bite → Slash → Spinning bite → Reset
Tank: Ground slam → Shoulder push → Heavy strike → Reset
Armored: Energy blast → Pulse → Multi-hit beam → Reset
Boss: Stomp → Claw × 2 → Energy AoE → Reset
Wind-up rendering in Game.ts renderZombies():

If windupTimer > 0: render scale/color shift (0.85-1.0 scale, +0.3 glow)
Add "charging" particle ring around zombie (visual tell for player)
🎨 Phase 5: Visual Polish & Details
Goal: Enhance perception of threat/tier/archetype.

Attack impact feedback in renderZombies():

Regular: red claw slash + blood mist
Fast: cyan dash trail + speed streaks
Tank: ground crack circle + dust cloud
Armored: energy pulse ring + electric arcs
Boss: multi-layer shockwave
Emit via game.effects.spawnXxx()
Tier aura emission in Game.update():

Tier 1: 1 particle/0.3s, Tier 2: 2/0.3s, Tier 3: 3/0.3s
Aura type (green mist, velocity trail, dust, electric arcs, multi-layer)
Stun animation:

Freeze limbs in current pose (no animation updates)
Render rotating stun stars around zombie, fade in/out
Hit recoil:

On damage: animationState = 'hit' (1-frame squeeze + white flash)
After 0.1s: return to previous state
